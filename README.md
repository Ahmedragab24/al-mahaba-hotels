# نظام المهابه لإدارة الحجوزات الفندقية (Al-Mahaba Hotels ERP)

نظام سحابي متكامل (B2B/B2C) مصمم خصيصاً لإدارة عمليات الشركات السياحية (Tour Operators/DMCs)، لضبط حجوزات الفنادق، إدارة الموردين، أتمتة محرك الأسعار، وإدارة دورة حياة الحجز من التسعير وحتى التسوية المالية.

---

## 🎯 الهيكل التشغيلي ومنطق العمل (Business Logic)

يعتمد النظام على ترابط عدة وحدات (Modules) تعمل معاً لتكوين دورة عمل متكاملة:

### 1. إدارة الموردين والعقود (Suppliers & Contracts)
- **بوابة الموردين (Supplier Portal):** يمكن للفنادق أو وكلاء السفر (Wholesalers / DMCs) التسجيل في النظام عبر شاشة تسجيل مخصصة.
- **الاعتمادات (Applications):** تخضع طلبات الموردين للمراجعة (قبول/رفض) من قبل الإدارة.
- **العقود (Contracts):** بعد اعتماد المورد، يتم ربطه بعقود سنوية أو موسمية تحدد شروط الدفع والتسهيلات.

### 2. محرك الأسعار والمواسم (Rates & Seasons Engine)
- **المواسم (Seasons):** تقسيم السنة إلى مواسم (عالي، منخفض، أعياد، الخ) وتطبيق الأسعار برمجياً بناءً على تاريخ وصول العميل.
- **إدارة الأسعار (Rates):** النظام لا يعتمد على سعر ثابت فقط، بل يحسب السعر بناءً على (الإشغال - Occupancy) وتكاليف المورد (Cost Price)، ليتم حساب سعر البيع (Selling Price) وهامش الربح (Margin) تلقائياً.

### 3. طلبات الأسعار للعملاء والموردين (Quotations & RFQs)
- **طلبات التسعير (RFQs):** في حال عدم وجود سعر متعاقد عليه، يُرسل قسم العمليات طلب تسعير (RFQ) للموردين للحصول على أفضل سعر متاح.
- **عروض الأسعار للعملاء (Quotations):** بناء عرض سعر متكامل للعميل يحتوي على الغرف، الضرائب، والتكلفة الإجمالية، بمدة صلاحية محددة. يمر عرض السعر بحالات: (مسودة -> بانتظار الموافقة -> معتمد -> مقبول من العميل -> منتهي).

### 4. إدارة الحجوزات (Bookings Management)
- بمجرد قبول العميل لعرض السعر، يتم تحويله تلقائياً إلى **حجز فعلي**.
- يمكن تتبع حالة الحجز (مؤكد، ملغى، قيد الانتظار) ومتابعة التواريخ الهامة (Deadlines) الخاصة بالإلغاء المجاني.

### 5. الإدارة المالية والضرائب (Finance, Invoices & Payables)
- **الفواتير وسندات القبض (Invoices & Receipts):** إصدار فواتير للعملاء وتسجيل الدفعات الواردة.
- **المدفوعات والمستحقات (Payables):** إدارة المبالغ المستحقة للموردين وجدولة سدادها بناءً على الفواتير المعتمدة.
- **الضرائب (Taxes):** وحدة مخصصة لتعريف الضرائب (مثل ضريبة القيمة المضافة) وتطبيقها تلقائياً على عروض الأسعار والفواتير.

### 6. سلسلة الاعتمادات وسجلات المراجعة (Approval Workflow & Audit)
- **مستويات الاعتماد (Approval Thresholds):** عند قيام موظف بتقديم خصم يتجاوز الحد المسموح له، يتم إيقاف العملية تلقائياً وتُرسل لمديره المباشر للاعتماد.
- **سجل المراجعة (Audit Log):** النظام يسجل **كل حركة** تحدث داخل النظام (من قام بإضافة/تعديل/حذف وما هو الوقت) لضمان الشفافية والمحاسبة (Accountability).

## 🛠 الهيكلة التقنية (Technical Architecture)

تم بناء النظام باستخدام أحدث التقنيات وأفضل الممارسات لضمان السرعة والتوسع:

- **إطار العمل للواجهات (Frontend):** 
  - `React.js` و `Vite` لتجربة تطوير وبناء فائقة السرعة.
  - `TanStack Start / Router` لإدارة توجيه الصفحات (Routing) والتحميل المسبق (Intent Preloading).
- **إدارة البيانات والحالة (Data Fetching & State):** 
  - `TanStack Query (React Query)` لجلب البيانات من الخادم وتخزينها محلياً (Caching) لتقليل الضغط على قاعدة البيانات وتسريع استجابة الواجهة.
- **التصميم وتجربة المستخدم (UI/UX):** 
  - `Tailwind CSS` بالإضافة إلى مكونات `Radix UI` المتطورة (عبر shadcn/ui) لتقديم واجهات احترافية وتدعم اللغة العربية بالكامل (RTL).
- **الخلفية وقاعدة البيانات (Backend & Database):** 
  - REST API (`VITE_API_URL`) مع PostgreSQL في الخادم.
  - المصادقة عبر JWT/token مع صلاحيات RBAC على مستوى التطبيق.
- **الاستضافة (Deployment):** 
  - يتم رفع النظام على `Vercel` باستخدام معمارية الـ Serverless (عبر مكتبة Nitro و Lovable Config) لضمان أداء عالٍ.

## 👥 نظام الصلاحيات والأدوار (Roles & Permissions)

النظام مبني على صلاحيات دقيقة جداً (RBAC) وتشمل:
1. **Super Admin / Admin:** وصول كامل للنظام والإعدادات وحذف/تعديل جميع البيانات.
2. **Sales Manager / Agent:** مسؤولون عن إنشاء عروض الأسعار (Quotations) ومتابعة العملاء والحجوزات.
3. **Operations Manager / Agent:** مسؤولون عن التواصل مع الموردين (RFQs) وتأكيد توافر الغرف.
4. **Finance Manager / Agent:** وصول مخصص للمالية (الفواتير، سندات القبض، والمدفوعات).
5. **Supplier:** وصول مقيد للمورد ليرى طلبات التسعير المرسلة إليه (RFQs) فقط ويقوم بالرد عليها.
6. **Viewer:** صلاحية قراءة فقط (للمراجعة أو الإدارة العليا).

## 🚀 كيفية تشغيل المشروع بيئة التطوير (Development Setup)

1. **المتطلبات الأساسية:** تثبيت Node.js (أو Bun).
2. **تثبيت الحزم:** 
   ```bash
   npm install
   ```
3. **تشغيل المشروع:**
   ```bash
   npm run dev
   ```
4. المتصفح سيفتح على: `http://localhost:5173`

*(تأكد من إعداد ملف `.env` وتعيين `VITE_API_URL` لعنوان الـ API قبل التشغيل)*.
